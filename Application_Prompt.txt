# Project Specification: ShopSphere Demo Backend

You are an experienced Senior Python Backend Engineer.

Your task is to build a **production-inspired Python backend** that will be used as a **QA testing playground** for multiple AI-powered Software Testing products.

This is NOT a tutorial project.
This is NOT a beginner CRUD application.
This is NOT an academic assignment.

The objective is to create a realistic backend that looks like something developed inside a software company.

---

## Primary Goal

The backend will be used for an AI project called:

**AI Regression Impact Analyzer**

The application itself will NOT be deployed.

Instead, it will be used to:

* Create realistic Git commits
* Generate Git diffs
* Simulate production code changes
* Test AI-based regression impact analysis
* Create multiple QA scenarios

Therefore, the project should contain meaningful business logic and realistic dependencies between modules.

---

# Business Rules

Create realistic rules.

Examples:

A coupon cannot be applied twice.

Inventory cannot become negative.

Users cannot checkout with an empty cart.

Payment cannot proceed without inventory validation.

Orders cannot be created without successful payment.

Cancelled orders cannot be shipped.

Reviews require completed purchases.

Session expires after inactivity.

Password policy should include validation.

Implement many similar realistic rules.

The project is intentionally built for AI testing.

Design code so that future commits can easily simulate:

Profile validation changes

Notification changes

Pricing changes

Order processing updates

Database migration simulations (without an actual database)

These scenarios will later be committed to Git and analyzed by an AI Regression Impact Analyzer.
